This is a comprehensive list covering fundamental and advanced concepts in machine learning and data analysis. Let's break down each one:

 

### Types of Regression Analysis Models

Regression analysis is a statistical method used to model the relationship between a dependent variable (target) and one or more independent variables (features). The goal is to predict continuous outcomes.

1.   Simple Linear Regression:  Models the relationship between two continuous variables (one independent, one dependent) using a straight line: $y = beta_0 + beta_1x + epsilon$.
2.   Multiple Linear Regression:  Extends simple linear regression to multiple independent variables: $y = beta_0 + beta_1x_1 + beta_2x_2 + ... + beta_nx_n + epsilon$.
3.   Polynomial Regression:  Models a curvilinear relationship by adding polynomial terms (e.g., $x^2, x^3$) to the linear model. It's still linear in terms of the coefficients, allowing linear model techniques to be used.
4.   Lasso Regression (L1 Regularization):  Adds a penalty equal to the absolute value of the magnitude of coefficients. It can shrink some coefficients to zero, performing feature selection and helping prevent overfitting.
5.   Ridge Regression (L2 Regularization):  Adds a penalty equal to the square of the magnitude of coefficients. It shrinks coefficients but doesn't set them exactly to zero, primarily used to reduce multicollinearity and prevent overfitting.
6.   Elastic Net Regression:  A hybrid of Lasso and Ridge regression, combining both L1 and L2 penalties. It's useful when there are multiple correlated features.
7.   Non-linear Regression:  Models where the relationship between dependent and independent variables is inherently non-linear and cannot be transformed into a linear form. Examples include exponential, logarithmic, or logistic functions fitted directly (not transformations for linear models).
8.   Quantile Regression:  Instead of modeling the mean of the dependent variable (as in OLS), it models the conditional median or other quantiles. Useful for understanding how predictors affect different parts of the response distribution.

 

### Cross Validation

Cross-validation is a resampling procedure used to evaluate machine learning models on a limited data sample. Its primary purpose is to assess how well a model will generalize to an independent dataset and to prevent overfitting.

*    Problem it solves:  Training and testing a model on the same data leads to an overly optimistic estimate of performance. A simple train/test split can be sensitive to the particular split chosen.
*    How it works (K-Fold Cross-Validation): 
    1.  The entire dataset is divided into `k` equally sized "folds" or subsets.
    2.  The model is trained `k` times. In each iteration:
        *   One fold is used as the validation (test) set.
        *   The remaining `k-1` folds are combined to form the training set.
    3.  The performance metric (e.g., accuracy, MSE) is recorded for each of the `k` iterations.
    4.  The final performance of the model is the average of the `k` recorded metrics.
*    Benefits: 
    *   Provides a more robust and reliable estimate of model performance than a single train/test split.
    *   Ensures that every data point gets to be in a test set exactly once, and in a training set `k-1` times.
    *   Helps detect and mitigate overfitting.
*    Variations:  Leave-One-Out Cross-Validation (LOOCV, where k=N, number of data points), Stratified K-Fold (ensures each fold has roughly the same proportion of class labels as the full dataset, crucial for imbalanced classification).

 

### Non-linear to Linear Models

This refers to techniques used to transform a dataset with non-linear relationships into a form where a linear model can be effectively applied. This is often done because linear models are simpler, faster, and more interpretable.

*    Core Idea:  Create new features or transform existing ones such that a linear combination of these (transformed) features can approximate the target variable.
*    Methods: 
    1.   Polynomial Feature Engineering:  Introduce higher-order terms of existing features. If $y = beta_0 + beta_1x + beta_2x^2 + epsilon$, this is non-linear in $x$ but linear in the parameters $beta_i$. You can create a new feature $z = x^2$ and then model $y = beta_0 + beta_1x + beta_2z + epsilon$.
    2.   Logarithmic Transformation: 
        *   If the relationship is exponential (e.g., $y = ae^{bx}$), taking the natural logarithm of $y$ yields $ln(y) = ln(a) + bx$, which is linear in $ln(y)$ and $x$.
        *   Can also be applied to skewed independent variables to make their distribution more symmetric, improving linearity assumptions.
    3.   Exponential Transformation:  Less common, but can be used if the dependent variable has a highly skewed distribution and you want to model its log.
    4.   Square Root/Reciprocal Transformations:  Can stabilize variance and make relationships more linear for certain types of data distributions.
    5.   Feature Engineering:  Creating entirely new features based on domain knowledge that capture non-linear interactions. For example, if a relationship depends on a ratio of two features, creating that ratio as a new feature.

 

### Principal Component Analysis (PCA)

PCA is a dimensionality reduction technique used to reduce the number of features in a dataset while retaining most of the variance.

*    Purpose: 
    *   Reduce the "curse of dimensionality" (when too many features hinder model performance).
    *   Speed up training algorithms.
    *   Visualize high-dimensional data (e.g., reducing to 2 or 3 components).
    *   Remove noise and improve data interpretability.
*    Core Idea:  It transforms the data into a new coordinate system where the new axes (called Principal Components) are orthogonal and capture the maximum variance in the data. The first principal component captures the most variance, the second the second most, and so on.
*    How it works (simplified): 
    1.   Standardize the data:  Scale features to have zero mean and unit variance.
    2.   Compute the covariance matrix:  This matrix shows how features vary with each other.
    3.   Calculate eigenvalues and eigenvectors:  Eigenvectors of the covariance matrix represent the principal components (directions of maximum variance), and eigenvalues represent the magnitude of variance along those components.
    4.   Select principal components:  Order eigenvectors by their eigenvalues (largest first) and select the top `k` components that explain a sufficient amount of variance (e.g., 95%).
    5.   Transform data:  Project the original data onto these selected `k` principal components.
*    Limitations: 
    *   The new components are linear combinations of original features, making them less interpretable.
    *   Assumes linearity in the data structure.
    *   Sensitive to scaling of features.

 

### Factor Analysis

Factor Analysis (FA) is a statistical method similar to PCA, primarily used for dimensionality reduction and understanding the underlying latent structure among a set of observed variables.

*    Purpose: 
    *   Identify underlying unobservable (latent) factors that explain the correlations among a set of observed variables.
    *   Reduce the number of variables by grouping them into factors.
    *   Understand the construct validity of a survey or measurement instrument.
*    Core Idea:  FA assumes that the observed variables are linear combinations of a few common factors (latent variables) and unique factors (errors). It tries to explain the observed covariance among variables in terms of these underlying factors.
*    Difference from PCA: 
    *    Goal:  PCA focuses on summarizing variance by finding orthogonal components that capture maximum variance. FA focuses on explaining covariance, assuming a generative model where latent factors cause observed variables.
    *    Assumptions:  FA makes stronger statistical assumptions (e.g., observed variables are normally distributed, common factors are uncorrelated with unique factors). PCA is purely a mathematical transformation.
    *    Output:  PCA components are computed directly from the data. FA factors are inferred and often rotated to improve interpretability.
*    Use Cases:  Psychology (identifying personality traits from survey responses), market research (understanding consumer preferences), social sciences.

 

### Logistic Regression

Despite its name, Logistic Regression is a statistical model used for  classification  tasks, specifically for predicting the probability of a binary outcome (e.g., yes/no, true/false, 0/1). It can be extended for multi-class classification.

*    Purpose:  Predict the probability that a given input instance belongs to a particular class.
*    How it works: 
    1.  It uses a linear equation (like linear regression) to calculate a "logit" score: $z = beta_0 + beta_1x_1 + ... + beta_nx_n$.
    2.  This logit score is then passed through a  sigmoid (logistic) function  to transform it into a probability value between 0 and 1: $P(Y=1|X) = frac{1}{1 + e^{-z}}$.
    3.  A threshold (e.g., 0.5) is applied to these probabilities to classify the instance into one of the two classes.
*    Benefits: 
    *   Outputs probabilities, which can be useful for decision-making.
    *   Coefficients are interpretable in terms of odds ratios.
    *   Relatively simple and efficient.
    *   Good baseline for binary classification problems.
*    Limitations: 
    *   Assumes a linear relationship between the independent variables and the log-odds of the dependent variable.
    *   Can be sensitive to multicollinearity.
    *   Not suitable for complex non-linear relationships.

 

### Support Vector Machines (SVM)

SVMs are powerful supervised learning models used for both classification and regression, but primarily known for classification.

*    Purpose:  To find an optimal hyperplane that best separates data points of different classes in a high-dimensional space.
*    Core Idea (for linear classification): 
    1.  Finds the hyperplane that maximizes the "margin" (the distance between the hyperplane and the nearest data points from each class).
    2.  The data points that lie closest to the hyperplane are called "support vectors" and are critical in defining the hyperplane.
*    Kernel Trick (for non-linear classification): 
    *   When data is not linearly separable in the original feature space, SVMs use the "kernel trick."
    *   A kernel function implicitly maps the input data into a higher-dimensional feature space where it might become linearly separable, without actually computing the coordinates in that higher dimension.
    *   Common kernels: Radial Basis Function (RBF), Polynomial, Sigmoid.
*    Benefits: 
    *   Effective in high-dimensional spaces.
    *   Relatively robust against overfitting, especially with a good margin.
    *   Versatile due to various kernel functions.
*    Limitations: 
    *   Can be computationally intensive for large datasets.
    *   Choosing the right kernel and tuning hyperparameters can be challenging.
    *   Less interpretable than models like decision trees.

 

### K-nearest neighbours (K-NN)

K-NN is a non-parametric, lazy learning algorithm used for both classification and regression.

*    Purpose:  Classify a new data point based on the majority class of its 'k' nearest neighbors in the feature space (for classification), or predict its value based on the average of its 'k' nearest neighbors (for regression).
*    How it works (Classification): 
    1.  When a new data point needs to be classified, it calculates the distance (e.g., Euclidean distance) between this new point and all existing points in the training dataset.
    2.  It identifies the 'k' closest data points (neighbors).
    3.  It assigns the new data point to the class that is most common among its 'k' neighbors (majority vote).
*    Key Hyperparameter:  `k` (the number of neighbors). The choice of `k` significantly impacts performance.
*    Benefits: 
    *   Simple to understand and implement.
    *   No explicit training phase (lazy learner).
    *   Can adapt to complex decision boundaries.
    *   Can be used for both classification and regression.
*    Limitations: 
    *   Computationally expensive during prediction, especially for large datasets (needs to calculate distances to all training points).
    *   Sensitive to irrelevant features and the scale of features (feature scaling is often necessary).
    *   Sensitive to outliers.
    *   Requires choosing an appropriate distance metric.

 

### Naïve Bayes Classifier

The Naïve Bayes classifier is a family of probabilistic classification algorithms based on  Bayes' theorem  with a "naïve" assumption of conditional independence between features given the class label.

*    Purpose:  To predict the class of a new data point based on the probabilities of its features.
*    Core Idea:  It calculates the probability of a data point belonging to a particular class ($P(text{Class}|text{Features})$) using Bayes' theorem:
    $P(text{Class}|text{Features}) = frac{P(text{Features}|text{Class}) times P(text{Class})}{P(text{Features})}$
    The "naïve" assumption is that all features are independent of each other given the class. This simplifies $P(text{Features}|text{Class})$ to the product of individual feature probabilities:
    $P(text{Features}|text{Class}) = P(F_1|text{Class}) times P(F_2|text{Class}) times ... times P(F_n|text{Class})$
*    Types: 
    *    Gaussian Naïve Bayes:  Assumes features follow a Gaussian (normal) distribution.
    *    Multinomial Naïve Bayes:  Suitable for discrete counts, often used in text classification (e.g., word counts).
    *    Bernoulli Naïve Bayes:  Suitable for binary/Boolean features (presence or absence of a feature).
*    Benefits: 
    *   Simple and fast to train, even with large datasets.
    *   Scalable with high-dimensional data.
    *   Often performs surprisingly well in text classification and spam filtering.
*    Limitations: 
    *   The "naïve" independence assumption is rarely true in real-world data, which can sometimes lead to suboptimal performance.
    *   Zero-frequency problem: If a category in the test data was not observed in the training data, its probability will be zero, leading to zero overall probability (handled by Laplace smoothing).

 

### Decision Tree Classification

A Decision Tree is a non-parametric supervised learning algorithm used for both classification and regression tasks. It builds a tree-like model of decisions based on feature values.

*    Purpose:  To classify data by creating a set of rules inferred from the features of the data.
*    How it works: 
    1.  The tree starts with a  root node  representing the entire dataset.
    2.  It splits the data into two or more subsets based on the feature that provides the "best" split (e.g., maximizes  information gain  or minimizes  Gini impurity ). The goal is to make the resulting subsets as homogeneous as possible with respect to the target variable.
    3.  This process is recursively repeated for each branch until a stopping criterion is met (e.g., maximum tree depth, minimum number of samples in a leaf node, or no further gain from splitting).
    4.  The final nodes are  leaf nodes , which contain the class label (for classification) or a predicted value (for regression).
*    Key Concepts: 
    *    Nodes:  Represent a test on an attribute.
    *    Branches:  Represent the outcome of the test.
    *    Leaves:  Represent the class label or predicted value.
    *    Splitting Criteria:  Information Gain (ID3, C4.5) or Gini Impurity (CART).
*    Benefits: 
    *   Easy to understand and interpret (visualizable).
    *   Requires little data preparation (can handle both numerical and categorical data, no scaling needed).
    *   Can handle multi-output problems.
*    Limitations: 
    *   Prone to overfitting, especially with complex trees.
    *   Can be unstable; small changes in data can lead to a completely different tree.
    *   Can create biased trees if classes are imbalanced.

 

### Random Forest Classification

Random Forest is an ensemble learning method for classification and regression that operates by constructing a multitude of decision trees during training and outputting the class that is the mode of the classes (classification) or mean prediction (regression) of the individual trees.

*    Purpose:  Improve predictive accuracy and control overfitting compared to a single decision tree.
*    Core Idea:  It leverages the power of "the crowd" by combining the predictions of many individual decision trees.
*    How it works (Two main randomizing mechanisms): 
    1.   Bagging (Bootstrap Aggregating): 
        *   It creates multiple subsets of the training data by  bootstrapping  (random sampling with replacement). Each subset is used to train a separate decision tree.
        *   This ensures that each tree sees a slightly different version of the data.
    2.   Feature Randomness: 
        *   When building each decision tree, instead of considering all features for the best split, Random Forest considers only a random subset of features at each node. This decorrelates the trees.
    3.   Aggregation: 
        *   For classification, the final prediction is determined by a  majority vote  among the individual trees.
        *   For regression, the average of the predictions from all trees is taken.
*    Benefits: 
    *   Reduces overfitting significantly compared to a single decision tree.
    *   Highly accurate and robust.
    *   Can handle large datasets and high dimensionality.
    *   Provides estimates of feature importance.
    *   Handles missing values automatically to some extent.
*    Limitations: 
    *   Less interpretable than a single decision tree (it's a "black box").
    *   Can be computationally more intensive and slower than a single decision tree, especially with many trees.
    *   Requires more computational resources and memory.

 

### Evaluating Classifier Models

Evaluating classifier models involves using various metrics to assess their performance, especially on unseen data. No single metric is universally best; the choice depends on the problem and business goals.

1.   Confusion Matrix:  A table that summarizes the predictions made by a classifier compared to the actual classes.
    *    True Positive (TP):  Correctly predicted positive class.
    *    True Negative (TN):  Correctly predicted negative class.
    *    False Positive (FP):  Incorrectly predicted positive (Type I error).
    *    False Negative (FN):  Incorrectly predicted negative (Type II error).
    |                 | Predicted Positive | Predicted Negative |
    | :    -- | :     -- | :     -- |
    | Actual Positive | TP                 | FN                 |
    | Actual Negative | FP                 | TN                 |

2.   Accuracy:  The proportion of correctly classified instances out of the total instances.
    *   Formula: $(TP + TN) / (TP + TN + FP + FN)$
    *    Use:  Good for balanced datasets.
    *    Issue:  Can be misleading for imbalanced datasets (e.g., 99% accuracy on a dataset with 99% negative class if it just predicts all negative).

3.   Precision (Positive Predictive Value):  The proportion of positive predictions that were actually correct.
    *   Formula: $TP / (TP + FP)$
    *    Use:  Important when the cost of a False Positive is high (e.g., medical diagnosis, spam detection).

4.   Recall (Sensitivity, True Positive Rate):  The proportion of actual positive cases that were correctly identified.
    *   Formula: $TP / (TP + FN)$
    *    Use:  Important when the cost of a False Negative is high (e.g., fraud detection, disease screening).

5.   F1-Score:  The harmonic mean of Precision and Recall. It provides a single score that balances both metrics.
    *   Formula: $2 times (text{Precision} times text{Recall}) / (text{Precision} + text{Recall})$
    *    Use:  Good when you need a balance between Precision and Recall, especially with uneven class distribution.

6.   ROC Curve (Receiver Operating Characteristic) & AUC (Area Under the Curve): 
    *    ROC Curve:  Plots the True Positive Rate (Recall) against the False Positive Rate (FP / (FP + TN)) at various classification thresholds. It shows the trade-off between sensitivity and specificity.
    *    AUC:  The area under the ROC curve.
        *   An AUC of 1.0 indicates a perfect classifier.
        *   An AUC of 0.5 indicates a classifier no better than random guessing.
    *    Use:  Excellent for evaluating models across all possible classification thresholds, robust to class imbalance.

7.   Log Loss (Cross-Entropy Loss):  Measures the performance of a classification model where the prediction is a probability value between 0 and 1. It heavily penalizes confident incorrect predictions.
    *    Use:  Preferred metric for models that output probabilities. Lower log loss is better.

 

### Clustering Methods

Clustering is an  unsupervised learning  technique that involves grouping a set of data points into subsets (clusters) such that data points within the same cluster are more similar to each other than to those in other clusters. Unlike classification, there are no predefined labels.

*    Purpose: 
    *   Discover natural groupings or patterns in data.
    *   Data compression (representing a cluster by its centroid).
    *   Anomaly detection (points far from any cluster).
    *   Customer segmentation, document analysis, image segmentation.
*    Key Methods: 

1.   K-Means Clustering: 
    *    Type:  Partitional clustering.
    *    How it works: 
        1.  Specify the desired number of clusters, `K`.
        2.  Randomly initialize `K` cluster centroids.
        3.   Assignment Step:  Assign each data point to the cluster whose centroid is closest to it.
        4.   Update Step:  Recalculate the centroids as the mean of all data points assigned to that cluster.
        5.  Repeat steps 3 and 4 until the centroids no longer move significantly or a maximum number of iterations is reached.
    *    Strengths:  Relatively simple, fast, and scalable for large datasets.
    *    Limitations:  Requires specifying `K` in advance, sensitive to initial centroid placement, struggles with non-spherical clusters and varying cluster densities, sensitive to outliers.

2.   Hierarchical Clustering: 
    *    Type:  Builds a hierarchy (tree) of clusters.
    *    Approaches: 
        *    Agglomerative (Bottom-Up):  Starts with each data point as its own cluster, then iteratively merges the two closest clusters until all points are in a single cluster or a stopping criterion is met.
        *    Divisive (Top-Down):  Starts with all data points in one cluster and recursively splits them into smaller clusters.
    *    Output:  A dendrogram, which is a tree-like diagram showing the sequence of merges or splits. You can then "cut" the dendrogram at a certain height to get a desired number of clusters.
    *    Strengths:  Does not require specifying `K` in advance, provides a full hierarchy of clusters, can handle arbitrary cluster shapes (depending on linkage method).
    *    Limitations:  Computationally more expensive (especially for large datasets), difficult to handle large datasets.

3.   DBSCAN (Density-Based Spatial Clustering of Applications with Noise): 
    *    Type:  Density-based clustering.
    *    How it works:  Groups together data points that are closely packed together, marking as outliers points that lie alone in low-density regions.
    *    Key Concepts: 
        *    Core Point:  A point that has at least `MinPts` (a parameter) neighbors within a distance `eps` (another parameter).
        *    Border Point:  A point that has fewer than `MinPts` neighbors but is within `eps` distance of a core point.
        *    Noise Point:  A point that is neither a core point nor a border point.
    *    Strengths:  Can find clusters of arbitrary shapes, doesn't require `K` in advance (implicitly determined by density), can identify noise/outliers.
    *    Limitations:  Struggles with varying densities in clusters, sensitive to parameters `eps` and `MinPts`, less effective for high-dimensional data.

4.   Gaussian Mixture Models (GMM): 
    *    Type:  Probabilistic clustering.
    *    How it works:  Assumes that data points are generated from a mixture of several Gaussian (normal) distributions with unknown parameters (mean, variance, mixing proportions). It uses an Expectation-Maximization (EM) algorithm to estimate these parameters.
    *    Output:  Unlike K-Means, GMM provides soft assignments, meaning each data point has a probability of belonging to each cluster.
    *    Strengths:  More flexible than K-Means (can model non-spherical clusters), provides probabilistic assignments, robust to noise.
    *    Limitations:  Can be computationally intensive, sensitive to initialization, requires specifying the number of components (clusters).

 

### Association Rule

Association rule mining is an unsupervised learning technique used to discover interesting relationships or associations between variables (items) in large datasets, often transactional databases.

*    Purpose:  To find rules that describe strong relationships between co-occurring items. A classic example is "market basket analysis," where the goal is to find items frequently bought together (e.g., "customers who buy milk and bread also tend to buy butter").
*    Key Concepts: 
    *    Itemset:  A collection of one or more items (e.g., {Milk, Bread}).
    *    Frequent Itemset:  An itemset that appears frequently enough in the dataset (i.e., its  support  is above a minimum threshold).
    *    Association Rule:  An implication expression of the form $X Rightarrow Y$, where $X$ and $Y$ are disjoint itemsets (i.e., $X cap Y = emptyset$). $X$ is called the  antecedent  (or LHS - Left Hand Side), and $Y$ is called the  consequent  (or RHS - Right Hand Side).
*    Metrics for Evaluating Rules: 
    1.   Support ($S(X Rightarrow Y)$): 
        *   Definition: The proportion of transactions in the dataset that contain both $X$ and $Y$.
        *   Formula: $P(X cup Y)$
        *   Importance: Indicates how frequently the itemset ${X, Y}$ appears together. A rule with low support might be due to chance.
    2.   Confidence ($C(X Rightarrow Y)$): 
        *   Definition: The conditional probability that $Y$ appears in a transaction given that $X$ has already appeared.
        *   Formula: $P(Y|X) = S(X cup Y) / S(X)$
        *   Importance: Indicates the reliability of the rule. A high confidence means that if $X$ is present, $Y$ is very likely also present.
    3.   Lift ($L(X Rightarrow Y)$): 
        *   Definition: Measures how much more likely itemset $Y$ is to be purchased when itemset $X$ is purchased, compared to the baseline likelihood of purchasing $Y$ independently.
        *   Formula: $C(X Rightarrow Y) / S(Y) = P(Y|X) / P(Y)$
        *   Importance:
            *    Lift = 1:  $X$ and $Y$ are independent (no association).
            *    Lift > 1:  Positive association (purchasing $X$ increases the likelihood of purchasing $Y$).
            *    Lift < 1:  Negative association (purchasing $X$ decreases the likelihood of purchasing $Y$).

 

### Apriori Algorithm with Example

The Apriori algorithm is a classic and foundational algorithm for mining frequent itemsets and discovering association rules from large transactional databases. It's based on the "Apriori property."

*    Apriori Property:  If an itemset is frequent, then all of its subsets must also be frequent. Conversely, if any subset of an itemset is infrequent, then the itemset itself cannot be frequent. This property is used to prune the search space.

*    Steps of the Apriori Algorithm: 

    1.   Generate Frequent 1-Itemsets (L1): 
        *   Count the occurrences of each individual item.
        *   Filter out items whose support (frequency) is below a minimum support threshold (`min_support`). The remaining items form L1.

    2.   Generate Candidate k-Itemsets (Ck) from (k-1)-Itemsets (Lk-1): 
        *   Join Lk-1 with itself to create candidate k-itemsets. For example, if L2 contains {A, B} and {A, C}, it would generate candidate {A, B, C}.

    3.   Prune Candidates (Apriori Property): 
        *   For each candidate k-itemset in Ck, check if all of its (k-1)-subsets are present in Lk-1.
        *   If any (k-1)-subset is *not* in Lk-1, then the candidate k-itemset is deemed infrequent and is removed from Ck.

    4.   Generate Frequent k-Itemsets (Lk): 
        *   Scan the entire transaction database to count the actual support for the remaining (pruned) candidate k-itemsets in Ck.
        *   Filter out candidates whose support is below `min_support`. The remaining form Lk.

    5.   Repeat Steps 2-4  until no more frequent itemsets can be found (i.e., Lk is empty).

    6.   Generate Association Rules: 
        *   Once all frequent itemsets are found, generate all possible non-empty consequent (Y) and antecedent (X) pairs for each frequent itemset.
        *   Calculate the  confidence  for each rule ($C(X Rightarrow Y) = S(X cup Y) / S(X)$).
        *   Filter out rules whose confidence is below a minimum confidence threshold (`min_confidence`).
        *   (Optional) Calculate  Lift  for the remaining rules to further assess their strength and interest.

*    Example: 

    Let's consider a transaction dataset:
    *   T1: {Milk, Bread, Butter}
    *   T2: {Milk, Bread}
    *   T3: {Milk, Eggs}
    *   T4: {Bread, Butter}
    *   T5: {Bread, Eggs}

    Set `min_support = 0.4` (meaning an itemset must appear in at least 2 out of 5 transactions).
    Set `min_confidence = 0.6`.

     

     Step 1: Generate Frequent 1-Itemsets (L1) 

    | Item  | Count | Support | Frequent? |
    | : - | : - | :   | :  -- |
    | Milk  | 3     | 0.6     | Yes       |
    | Bread | 4     | 0.8     | Yes       |
    | Butter| 2     | 0.4     | Yes       |
    | Eggs  | 2     | 0.4     | Yes       |

    `L1 = {{Milk}, {Bread}, {Butter}, {Eggs}}`

     

     Step 2: Generate Candidate 2-Itemsets (C2) from L1 
    (Join L1 with L1: all possible pairs from L1)

    `C2 = {{Milk, Bread}, {Milk, Butter}, {Milk, Eggs}, {Bread, Butter}, {Bread, Eggs}, {Butter, Eggs}}`

     

     Step 3 & 4: Prune C2 and Generate Frequent 2-Itemsets (L2) 

    Scan transactions to count support for C2:

    | Itemset         | Count | Support | Frequent? |
    | :    -- | : - | :   | :  -- |
    | {Milk, Bread}   | 2     | 0.4     | Yes       |
    | {Milk, Butter}  | 1     | 0.2     | No        |
    | {Milk, Eggs}    | 1     | 0.2     | No        |
    | {Bread, Butter} | 2     | 0.4     | Yes       |
    | {Bread, Eggs}   | 1     | 0.2     | No        |
    | {Butter, Eggs}  | 0     | 0.0     | No        |

    `L2 = {{Milk, Bread}, {Bread, Butter}}`

     

     Step 2 (for k=3): Generate Candidate 3-Itemsets (C3) from L2 

    Join L2 with L2. The only way to form a 3-itemset is if the first (k-1) items are common.
    `{Milk, Bread}` and `{Bread, Butter}` share 'Bread' but the others don't match for a clean join into a larger itemset (e.g., they don't overlap to produce a new unique 3-itemset where all subsets are in L2 using the formal Apriori join step). In this simplified example, no new candidates of length 3 can be generated from L2.

    So, `C3` is empty. The algorithm stops generating frequent itemsets.
